# piper-metawork

Using the metawork MQTT protocol, this project sends the joint angles
generated by the ik-cd-worker to the real PiPER robot.


metawork MQTT protocolを使ってik-cd-workerが生成するPiPERのジョイント
角を実ロボットのPiPERに送る

以下の外部リポジトリを利用する
1. ブラウザ上のVR環境でPiPERを操作するためのAFrameベースのWebアプリケーションのリポジトリ[robot-loader](https://github.com/TSUSAKA-ucl/robot-loader)および[ik-cd-worker](https://github.com/TSUSAKA-ucl/ik-cd-worker)
2. Metawork用JavaScriptライブラリーのパッケージ[`MetaworkMQTT/frontend-lib/`](https://github.com/TSUSAKA-ucl/MetaworkMQTT/tree/dev-wss/frontend-lib)
3. `MetaworkMQTT/frontend-lib/`を使った[AFrameコンポーネント`@ucl-nuee/metawork-aframe`](https://github.com/TSUSAKA-ucl/metawork-aframe/tree/main)
4. MQTTブローカーおよびMetaworkMQTT managerを立ち上げる[Docker composeファイル](https://github.com/TSUSAKA-ucl/Web-Based-VR-Teleoperation/tree/deskmeet/Docker)
5. エッジ側のPiPERロボット(or CoppeliaSim)のコントロール。コントロールとPiPERのCoppeliaSimのシミュレーションを立ち上げるDocker composeファイルは[こちら](https://github.com/TSUSAKA-ucl/Web-Based-VR-Teleoperation/tree/deskmeet/Robot_Control/SimDocker)

現状、4.5.は[`vettayruu/Web-Based-VR-Teleoperation`](https://github.com/vettayruu/Web-Based-VR-Teleoperation)のフォークで[同一githubリポジトリ](https://github.com/TSUSAKA-ucl/Web-Based-VR-Teleoperation/tree/deskmeet)の中にあるが、将来的には2.と4.を同一リポジトリで管理してロボット依存の5.は独立したリポジトリとして切り出したほうが良いと考えている。プロトコルが安定してきたら、3.も2.,4.と同一リポジトリに統合を検討する。
